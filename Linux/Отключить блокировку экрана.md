## Отключить блокировку экрана

Чтобы отключить блокировку экрана в KDE Plasma на Debian 12, можно  воспользоваться настройками системы или отредактировать конфигурационные файлы. Вот основные способы:

## Через системные настройки

1. Откройте «Системные настройки» (System Settings) через меню «Пуск» или сочетание клавиш.
2. Перейдите в раздел **«Экранная заставка»** (Screen Locking) в категории «Рабочий стол и поведение» (Workspace Behavior).
3. Отключите опцию **«Автоматически включать экранную заставку»** (Enable Screen Locking).
4. Если хотите настроить время до блокировки, в том же разделе найдите параметр **«Через сколько минут бездействия блокировать экран»** (Lock screen after) и выберите **«Никогда»** (Never) или установите нужное значение.
5. Проверьте настройки энергосбережения в разделе **«Энергосбережение»** (Power Management), чтобы убедиться, что там нет опций, связанных с блокировкой экрана.
6. Перезапустите систему или выйдите из учётной записи и войдите снова, чтобы изменения вступили в силу. 

## Через конфигурационный файл

Файл настроек блокировщика экрана находится по пути `~/.config/kscreenlockerrc`. Откройте его в текстовом редакторе и измените параметры в секции `[Daemon]`:

```
ini
[Daemon]
Autolock=false
LockOnResume=false
```

- `Autolock=false` отключает автоматическую блокировку экрана.
- `LockOnResume=false` предотвращает блокировку после выхода из спящего режима.

После сохранения файла выполните команду для применения изменений:

```
bash
qdbus org.kde.screensaver /ScreenSaver configure
```

## Через командную строку

Можно использовать утилиту `kwriteconfig5` для изменения настроек:

```
bash
kwriteconfig5 --file kscreenlockerrc --group Daemon --key Autolock --type bool false
kwriteconfig5 --file kscreenlockerrc --group Daemon --key LockOnResume --type bool false
```

Затем примените изменения командой:

```
bash
qdbus org.kde.screensaver /ScreenSaver configure
```

## Дополнительные рекомендации

- Если блокировка экрана продолжает срабатывать, проверьте, нет ли сторонних приложений или скриптов, которые её активируют.
- Убедитесь, что в настройках энергосбережения (Power Management) не заданы параметры, конфликтующие с настройками блокировщика.
- Если используете скрипты или приложения, которые могут влиять на активность  сеанса, убедитесь, что они корректно имитируют пользовательскую  активность (например, через `qdbus org.kde.screensaver /ScreenSaver SimulateUserActivity`).

Если проблема сохраняется, уточните, какие дополнительные настройки или приложения могут влиять на блокировку экрана.
